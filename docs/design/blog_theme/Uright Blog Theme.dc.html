Running Claude Code for Free with Nvidia NIM

Run the Claude Code CLI at no cost by connecting it to NVIDIA NIM’s free-tier inference microservices through a LiteLLM proxy.

Jack Wong {{ i.clock }}7 min read {{ i.link }}Share

The situation

If you’re hitting your Claude usage limits mid-session, or you’re a CS student who wants to try Claude Code without a paid subscription, this is for you. NVIDIA offers free API access to a catalog of open-source LLMs through build.nvidia.com — their hosted NIM platform.

By placing a LiteLLM proxy in front of NVIDIA NIM, you can point Claude Code at these models. LiteLLM handles translation between Anthropic’s Messages API and the OpenAI-compatible API that NIM speaks.

{{ i.warn }}

Fair warning. Claude Code will be talking to open-source models, not Claude. Extended thinking and complex tool-use chains may not work reliably — basic chat, code generation, and single-file edits generally work fine.

Set up the LiteLLM proxy

Create an empty folder and add a config.yaml:

{{ i.fileText }} config.yaml yaml {{ i.copy }}copy 
model_list:
  - model_name: claude-sonnet-4-6
    litellm_params:
      model: nvidia_nim/qwen/qwen3.5-122b-a10b
      api_key: os.environ/NVIDIA_NIM_API_KEY

  - model_name: claude-opus-4-6
    litellm_params:
      model: nvidia_nim/z-ai/glm5
      api_key: os.environ/NVIDIA_NIM_API_KEY

litellm_settings:
  drop_params: true  # drop Anthropic-only params

Pick your models

The NIM catalog has 40+ models. Some strong options for coding tasks:

Model NIM ID Notes
Qwen 3.5 122Bqwen/qwen3.5-122bStrong coder, MoE
GLM-5z-ai/glm5744B MoE, great reasoning
Kimi K2.5moonshotai/kimi-k2.51T MoE from Moonshot

Run the proxy

{{ i.terminal }} bash {{ i.copy }}copy 
$ docker run -d -p 4000:4000 \
    -e NVIDIA_NIM_API_KEY="nvapi-..." \
    -v $(pwd)/config.yaml:/app/config.yaml \
    --name litellm-nim \
    docker.litellm.ai/berriai/litellm:main-stable

Reload your shell, run claude-nim, and you have a free, no-subscription fallback for Claude Code.
